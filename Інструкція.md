# Інструкція

Підключення домену **`time-to-fight.com.ua`** без `www` до GitHub Pages (основна адреса: `https://time-to-fight.com.ua`).

---

## Крок 1. DNS у реєстратора домену

1. Увійти в кабінет, де куплений домен.
2. Відкрити керування доменом `time-to-fight.com.ua` → розділ **DNS / DNS-записи / Редактор DNS**.
3. Якщо є старі **A**-записи для **кореня** домену (`@` або порожнє ім’я) на чужі IP — видати або замінити на наведені нижче.
4. Додати **чотири** записи **типу A** для **кореня** домену:

| Тип | Ім’я / Host | Значення (IP) |
|-----|-------------|---------------|
| A | `@` або порожньо (як у панелі для кореня) | `185.199.108.153` |
| A | те саме | `185.199.109.153` |
| A | те саме | `185.199.110.153` |
| A | те саме | `185.199.111.153` |

5. Зберегти зміни.

Час поширення DNS — від хвилин до кількох годин (інколи до 24–48 год).

Актуальні IP для GitHub Pages: [документація GitHub](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site#configuring-an-apex-domain).

---

## Крок 2. Репозиторій на GitHub

1. Відкрити репозиторій з сайтом (наприклад **Hruzduk/Yorik**).
2. **Settings** → **Pages**.
3. У **Custom domain** ввести: **`time-to-fight.com.ua`** (без `https://`).
4. Зберегти.
5. Дочекатися успішної перевірки DNS.
6. Увімкнути **Enforce HTTPS**.

---

## Крок 3. Файл `CNAME` і деплой через Actions

Щоб кастомний домен **не зникав** після наступного деплою з GitHub Actions:

1. У **корені** репозиторію створити файл **`CNAME`** з одним рядком:

   ```text
   time-to-fight.com.ua
   ```

2. У `.github/workflows/pages.yml` у кроці збірки додати копіювання: `cp CNAME _site/` (поруч із копіюванням `index.html` тощо).
3. Закомітити й запушити в **`master`**.

---

## Крок 4. Перевірка

- Відкрити `https://time-to-fight.com.ua`.
- При проблемах — **Settings → Pages** і перевірка DNS-записів.
